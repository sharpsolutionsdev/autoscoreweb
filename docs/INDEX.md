# DartVoice Documentation Index

Welcome to the central documentation repository for the DartVoice ecosystem. This library is split into two halves: the business/marketing guides (focusing on strategy and growth) and the technical architecture (focusing on the code execution).

## 📚 Technical Manuals
These documents outline how the clients, websites, and billing servers interact.

1. [00_TECH_STACK_EXPLAINER.md](./tech/00_TECH_STACK_EXPLAINER.md): A simplified breakdown of the programming languages, frameworks, and third-party services powering the product.
2. [01_ARCHITECTURE.md](./tech/01_ARCHITECTURE.md): The overarching system pipeline and how audio is transformed into automated system clicks.
2. [02_WEB_AND_BILLING.md](./tech/02_WEB_AND_BILLING.md): A guide to the HTML static frontend, the Stripe webhook funnel, and the passwordless OTP bridge.
3. [03_CLIENT_APPS.md](./tech/03_CLIENT_APPS.md): A detailed breakdown of the three core products: the Windows `customtkinter` desktop builder, the Android Kivy mobile app, and the Manifest V3 Chrome Extension.
4. [04_DEPLOYMENT_GUIDE.md](./tech/04_DEPLOYMENT_GUIDE.md): Compilation commands for `.exe` (Pyinstaller), `.aab` (Buildozer), and `.zip` extensions.

## 💼 Business & Staff Guides
These documents outline our market positioning, monetization limits, and the growth/acquisition funnel.

1. [00_STAFF_GUIDE.md](./00_STAFF_GUIDE.md): The top-level introduction for internally onboarding team members.
2. [01_GENERAL_OVERVIEW.md](./01_GENERAL_OVERVIEW.md): Extended thoughts on the state of manual scoring in the darts market and DartVoice's solution.
3. [02_VALUE_PROPOSITION.md](./02_VALUE_PROPOSITION.md): Core marketing tenets and identifying the "frictionless play" ideal.
4. [03_PAYMENT_AND_FUNNEL.md](./03_PAYMENT_AND_FUNNEL.md): Outline of the SaaS recurring model.
5. [04_ADVERTISING_AND_MARKETING.md](./04_ADVERTISING_AND_MARKETING.md): Strategy around User Generated Content, Influencer sponsorships, and SEO positioning.
6. [05_UI_ENGINEERING_GUIDE.md](./05_UI_ENGINEERING_GUIDE.md): Legacy UI engineering documentation on integrating and bridging Python GUI libraries with Tailwind mockups.
7. [06_FUTURE_VISION_AND_MMR.md](./06_FUTURE_VISION_AND_MMR.md): Our roadmap toward ditching classic pay-to-enter tournaments for an Elo/MMR ranked matchmaking system.
8. [07_TEAM_COLLABORATION_GUIDE.md](./07_TEAM_COLLABORATION_GUIDE.md): Best practices for utilizing AI tools, maintaining safe Git environments, and dividing responsibilities with a back-end programmer.
